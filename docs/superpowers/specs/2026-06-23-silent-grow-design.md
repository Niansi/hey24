## 1. 产品定位与目标用户

### 1.1 产品定位
macOS 桌面视频剪辑工具。选择模板后上传照片，自动生成剪辑视频。

### 1.2 目标用户
- **新手用户：** 一键生成，无复杂参数
- **进阶用户：** 可展开高级选项，自定义分辨率、帧率、时长、背景音乐

### 1.3 MVP 范围

| 必须 (MVP) | 延后 |
|---|---|
| 「悄悄长大」模板 | 更多模板 |
| YuNet 人脸检测 + 5点关键点对齐 | 侧脸/遮挡场景增强 |
| 竖屏 1080×1920 输出（默认，可配置） | 横屏、正方形 |
| 硬切转场（0.2s/张） | 渐变、滑动等转场 |
| 单脸自动选 / 多脸用户选 | 智能自动选主脸 |
| 导出 MP4 (H.264) | 更多编码器/格式 |
| 基础背景音乐叠加 | 音频剪辑、多轨 |
| Vue 3 前端界面 | 主题/皮肤 |

## 3. 系统架构

```
┌─────────────────────────────────────────────────────┐
│                  Tauri 桌面壳                         │
│  ┌──────────────────┐  ┌──────────────────────────┐ │
│  │   前端 (Vue 3)    │  │   Rust 后端 (核心引擎)     │ │
│  │  · 模板选择        │  │  ① 图片导入模块           │ │
│  │  · 照片管理        │  │  ② 人脸检测模块 (YuNet)   │ │
│  │  · 人脸确认 UI      │  │  ③ 面部对齐模块           │ │
│  │  · 参数配置        │  │  ④ 帧序列生成             │ │
│  │  · 导出进度        │  │  ⑤ FFmpeg 调度 (子进程)   │ │
│  └──────────────────┘  └──────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

### 3.1 数据流（「悄悄长大」模板）

1. 用户拖入 N 张照片 → 前端显示缩略图列表
2. 点击「分析人脸」→ Rust 后端加载 YuNet ONNX 模型 → 逐张检测人脸 + 5 点关键点
3. 返回每张图的人脸框 + 关键点信息 → 前端展示
4. 单脸图片：自动选中；多脸图片：高亮所有脸，用户点击选择目标人
5. Rust 端「面部对齐」：计算相似变换矩阵 → 旋转、缩放、平移整图 → 双眼对齐水平线且间距统一
6. Rust 端生成帧序列 → FFmpeg 接收帧流水线 + 背景音乐 → 输出 MP4
7. 返回输出路径 → 前端展示「完成」，可预览/分享

## 4. 核心模块设计

### 4.1 人脸检测模块 (`engine/face`)

- **模型：** [YuNet](https://github.com/opencv/opencv_zoo/tree/main/models/face_detection_yunet) ONNX 格式
- **运行时：** Rust `ort` (ONNX Runtime) 加载推理
- **输入：** 图片（RGB，任意尺寸）
- **输出：** 人脸列表，每项包含：
  - `bbox`: [x, y, w, h] 人脸框（归一化坐标）
  - `confidence`: 检测置信度 (0-1)
  - `landmarks`: [[x, y]×5] 5点关键点（左眼中心、右眼中心、鼻尖、左嘴角、右嘴角）

### 4.2 面部对齐模块 (`engine/align`)

- **目标：** 让所有照片中的人物面部在画面中位置、大小、角度一致
- **算法流程：**
  1. 通过双眼中心计算双眼连线角度 → 旋转整图使双眼水平
  2. 以目标眼距（320px）为基准，计算缩放比例
  3. 以双眼目标位置（画面 38% 高度处）为中心，计算平移量
  4. 构建 2×3 相似变换矩阵（旋转 + 缩放 + 平移）
  5. 应用变换到整张图片
  6. 裁剪/填充至目标分辨率

- **边缘处理策略 (`fill_mode`):**
  - `blur_expand`（默认）：对边缘进行模糊扩展填充，避免黑边，尽量保留画面内容
  - `black`: 纯黑边填充
  - `crop`: 直接裁剪，不填充

### 4.3 渲染模块 (`engine/render`)

- **帧序列生成:** Rust 将对齐后的图片写入临时 PNG 序列
- **FFmpeg 调度:** 通过 `std::process::Command` 调 FFmpeg 子进程
- **FFmpeg 参数 (默认竖屏):**
  ```bash
  ffmpeg -framerate 5 -i frame_%04d.png \
    -i bgm.mp3 -shortest \
    -c:v h264_videotoolbox -b:v 8M \
    -vf "fps=30,format=yuv420p" \
    -pix_fmt yuv420p output.mp4
  ```
- 每张图 0.2s → 输入帧率 5fps，输出帧率 30fps（每帧重复 6 次）

### 4.4 模板引擎 (`engine/template`)

- **模板定义格式 (YAML):**
  ```yaml
  name: "悄悄长大"
  id: silent-grow
  version: 1
  output:
    default:
      width: 1080
      height: 1920
      fps: 5
      duration_per_photo: 0.2
  pipeline:
    - stage: face_detect
      model: yunet
      keypoints: true
    - stage: face_align
      target_eye_y: 0.38
      target_eye_distance: 320
      fill_mode: blur_expand
    - stage: render
      transition: cut
      fps: 30
      codec: h264_videotoolbox
  ```
- **扩展新模板：** 新增 YAML 文件 + 必要时新增 pipeline stage 实现

### 4.5 Rust 模块结构

| 模块 | 职责 |
|---|---|
| `engine/face` | 加载 ONNX、推理、返回脸框 + 关键点 |
| `engine/align` | 相似变换矩阵计算、图片旋转缩放平移 |
| `engine/render` | 帧序列生成、FFmpeg 子进程调度 |
| `engine/template` | 模板加载、管线编排 |
| `commands` | Tauri IPC 命令（供前端调用） |

## 5. 多脸 & 无脸场景处理

### 5.1 多脸图片
- 前端缩略图右上角显示 👥 标记
- 用户点击图片 → 弹出放大视图，所有人脸用矩形框标出
- 用户点击一个人脸框选定目标
- 未来可加自动预选策略（按人脸面积、中心偏移量、置信度加权排序）

### 5.2 无脸图片
- 缩略图标记 ⚠️ 提醒
- 默认退化方案：等比缩放 + 居中裁剪
- 用户可右键手动框选一个区域作为焦点

## 6. 前端界面

### 6.1 布局（三栏）

```
┌──────────────────────────────────────────────────┐
│  🎬 Hey24                         模板: 悄悄长大  │
├──────────┬──────────────────────┬────────────────┤
│  模板列表 │     照片时间线        │   预览窗口      │
│ ·悄悄长大 │  [图1][图2][图3]...  │  ┌──────────┐  │
│ ·(更多..) │  ·拖入排序 / 删除     │  │  竖屏预览  │  │
│          │  ·每张可点开确认人脸  │  │          │  │
│ ──────── │                      │  │          │  │
│  输出设置 │                      │  │          │  │
│ ·分辨率   │                      │  └──────────┘  │
│ ·时长    │  底部: [分析人脸]      │  ▶ 生成视频     │
│ ·帧率    │       [一键生成]      │                │
│ ·背景音乐 │                      │                │
├──────────┴──────────────────────┴────────────────┤
│  状态栏: 就绪 / 分析中... / 渲染中...               │
└──────────────────────────────────────────────────┘
```

### 6.2 交互流程
1. **选模板** → 左侧点击（默认「悄悄长大」）
2. **导照片** → 中间区域拖入/选择 N 张照片，支持拖拽排序
3. **确认人脸** → 点击「分析人脸」→ 单人脸自动通过，多人脸用户选择 → 缩略图显示确认标记
4. **调参数（可选展开）** → 右侧：切换分辨率/帧率/时长/背景音乐
5. **一键生成** → Rust 处理 + FFmpeg 渲染 → 进度条 → 完成预览
6. **导出** → 保存 MP4 到本地

## 9. 非功能需求

- **性能：** 单张图片人脸检测 < 50ms（CPU），50 张照片总处理 < 5s
- **离线：** 所有功能本地运行，无需联网
- **错误处理：** FFmpeg 失败时用户友好提示；大尺寸图片自动降采样
