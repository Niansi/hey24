🎬 三种架构方案对比

macOS 视频剪辑工具 - 选择整体架构

A

Rust 全栈方案

Rust 负责一切：ort 跑人脸检测、image 处理图片、ffmpeg-next 合成视频。前端只做 UI 壳。

👍 优点

  • 单一编译产物，无外部依赖
  • 性能极致，所有处理在 Rust 层

👎 缺点

  • ffmpeg-next 绑定复杂易出错
  • 调试视频编码问题困难
B

Rust + FFmpeg 管道 ⭐ 推荐

Rust 负责人脸检测 + 图片处理，视频合成委托给系统 FFmpeg（子进程）。

👍 优点

  • FFmpeg 管道成熟稳定
  • 人脸检测用 ort CPU 上飞快
  • 无需额外运行时依赖
  • 容易接入更多模板

👎 缺点

  • 用户需安装 FFmpeg（可内置）
  • 图片→视频需中转帧序列
C

Rust + Python 侧车

Rust 做 Tauri 壳，Python 侧车用 OpenCV 做检测+渲染。

👍 优点

  • Python 生态最丰富
  • InsightFace 等人脸方案随便换

👎 缺点

  • 需捆绑 Python 运行时（~50MB）
  • 进程间通信复杂
  • macOS 打包麻烦

💡 推荐 B：Rust + FFmpeg 管道
理由：① 够快——YuNet 单张图片 ~20ms，批处理无压力 ② 够稳——FFmpeg 的 filter_complex 管道是行业标准 ③ 够轻——ONNX 模型 ~300KB，不拖累打包体积 ④ 易扩展——新增模板只需调整 Rust 端图片处理逻辑
